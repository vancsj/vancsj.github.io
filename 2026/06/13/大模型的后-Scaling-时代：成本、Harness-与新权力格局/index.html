大模型的后 Scaling 时代：成本、Harness 与新权力格局

发表于 2026-06-13 阅读次数：

在2026年，scaling law依然成立：Anthropic 最新的Mythos/Fable模型，据说参数规模达到10T；Gemini参数量预计3-5T；国产大模型也纷纷达到1T规模。与规模对应的，是模型智能水涨船高。

那么问题来了：scaling law还能继续吗？我毫不怀疑100T的模型会更加智能，但有用得起的应用场景吗？

成本的墙

2026 年出了两个新闻，把”成本墙”从概念变成了现实。

Uber 四个月烧光全年 AI 预算。 Uber 设了个内部排行榜，按团队用 AI 工具的量排名。工程师们积极响应，大量使用 Claude Code 和 Cursor——然后发现全年预算在第一季度就花光了。Uber 不得不在六月对每个工具设了 $1,500/人/月的硬上限。COO Andrew Macdonald 在采访里说，token 消耗量和有用功能之间并没有清晰的对价关系。

微软在内部替换第三方模型。 Azure 现在每分钟处理 100 亿个 token（同比涨了 52 倍），AI 业务年化收入 $370 亿——但同时，微软在 Experiences + Devices 部门里逐步砍掉 Claude Code 许可证，把工程师往 GitHub Copilot CLI 上赶。原因很简单：自家的东西不用按 token 付费。GitHub Copilot 也从 2026 年 6 月开始转向按用量计费。

两个故事指向同一个结构性问题：agent 类 AI 的 token 消耗量跟前一代的 API 调用完全不是一个数量级。Goldman Sachs 预测 agentic AI 会让 token 消费量在 2030 年增长 24 倍，达到每月 120 quadrillion。OpenAI 内部最大的一个客户每月消耗 1000 亿 token，有单个员工一周就用掉 2100 亿。

Claude Code 把这个矛盾放到了最刺眼的位置。$200/月的 Max 20x 含几千刀的等价算力，中度使用绰绰有余。但用 API 按量计费做主力开发，要把一位 senior 工程师提效 30%，折算下来的费用已经逼近他的月薪。

API 单价在跌（GPT-4 到 Gemini 3.1 Flash 三年跌了 99.7%），但总消耗量涨得更快，意味着用户的账单总额不降反升。Gartner 预测 2030 年推理成本会比 2025 年降 90%，但也明确说：更便宜的 token 不会带来更便宜的企业 AI——因为 agent 类模型每任务需要的 token 远多于标准模型，而且供应商不会把成本降幅全部传导给用户。

这就是”成本的墙”的真实含义：scaling law 没有变。但用户侧的账单，开始让 CFO 皱眉了。

Agent 的爆发式增长带来了一个矛盾：应用对模型智能下限的容忍度在提高——GPT-5.5 比前代强，但 DeepSeek V4 Pro 在很多场景下已经”足够好”了。真正卡脖子的，是对低幻觉率的要求反而更高了。因为 agent 是自主行动的，它的错误不是”抛个异常等下次”，而是直接写坏代码、算错账、签错合同。

幻觉：可用性的第一条红线

说个场景：你让人写代码，他写错了——编译器会告诉你。你让人写合同，他引用了一条不存在的法条——谁来告诉你？

低幻觉率是所有应用场景中可用性的前提条件。 没有低幻觉率，其他都是空谈。

为什么？因为高幻觉率的模型在任何一个需要信任的场景里都不可用。客服算错退款金额，代码审查漏掉安全漏洞，法务分析引用不存在的判例——每个错误都是一次信任崩塌。用户只需要几次”被坑”的经历，就会彻底放弃这个工具。

幻觉率本质上是个注意力问题。上下文越长，注意力越分散，幻觉率越高。Fable 5 在 1M 上下文表现出极低幻觉率，但那是靠它的参数量硬撑。按这个比例推算，它的极限上下文可能在 5-10M。一旦推到那个级别，注意力稀疏了，幻觉率大概率飙升。这是注意力机制的内在秉性。

编程场景率先跑通，很大程度上因为它的 verification loop 是天生的：编译器和测试用例就是完美的幻觉检测器。代码不对就是不对，模型没法狡辩。客服、法务、医疗可没这种东西。

高幻觉率的模型会被市场直接剔除。份额减少都是轻的，用户直接不用了——就像早期的搜索引擎如果返回大量垃圾结果，你会直接换掉它。

幻觉说到底是个模型内在的问题，不能靠训一个更大的模型根除。但你可以绕过去：用工具来补充。比如代码场景，你让模型写段 SQL，它写错了——跑一下就能发现。这跟模型强不强关系不大，取决于两个条件：一是工具能提供验证能力，二是模型得主动去调用这些工具。

工具调用：模型能力的乘数

一个 70B 模型，允许 tool calling 之后的表现，在某些任务上能压过不让调工具的 1T 闭源模型。

2026 年，MCP已经成了连接模型和工具的”USB-C”。上万个公开 MCP 服务器，月均 SDK 下载量破亿。一个模型只要支持 MCP，就能瞬间获得搜索、代码执行、数据库查询、文件操作的能力。工具不再是定制集成，而是即插即用。

关键在于主动性——模型自己判断”这里我需要更多信息”，主动去查。能不能做到这一点，是模型能力上限的分水岭。

Harness 成为主战场

新的注意力机制——Mamba、NSA、MLA——正在大幅降低推理成本。但代价是模型的”脾气”更难摸透了。同样的架构，换一个任务可能表现天差地别。复杂度从模型层转移到了 harness 层。

Harness 是什么？是模型之外的整套工程系统：

  • 提示词管理和路由
  • 工具定义和编排（MCP 正在成为标准层）
  • 上下文窗口的策略性使用
  • 输出验证和纠错
  • 多模型间的调度

无论什么级别的模型——本地 8B 还是云端 Fable 5——没有良好的 workflow 定义和合适的 harness 管理，都不可能在实际场景中稳定输出。

Anthropic对OpenAI的市值反超已经证明了，光做好大模型不足以成为护城河。

Harness 领域已经形成了几个代表。Claude Code 是最成功的编程 harness，单产品年化收入达 $25 亿，支持子代理、hooks、MCP 集成和调度任务。OpenClaw 是 MIT 许可的开源个人 AI 助手，Node.js 构建，自带 100 多个 AgentSkills。Hermes Agent 是 Nous Research 在 2026 年 2 月发布的自改进 agent，十周内拿到 11 万 GitHub stars，核心差异是持续的学习闭环。

但它们都还处于粗放阶段：AI 自行为自己创建 skills 和 memory，缺少创建 harness 本身的 meta 工具。我们有 agent，但没有”制造 agent 的工具”。

Harness 方向会怎么走？我猜至少三个方向同时推进：框架层的公司（Anthropic Managed Agents 已经是了）、云厂商的托管服务、专注垂直场景的方案。人的角色也在跟着变——主要工作不再是写代码或调模型，而是设计和验证 harness，再验证 agent 的输出。你从”建造者”往”质检员”的方向走。

架构范式：本地小 + 远端大

Qwen 3.5 9B、Llama 3 8B 这一代模型，在 Apple Silicon M3 上跑 Q4 量化，生成速度约 18 tok/s，短提示首次响应 1-3 秒。推理成本几乎为零，隐私数据不出本机。2026 年，本地模型终于不再是玩具。

这就催生了一个新架构：本地小模型处理日常、延迟敏感、隐私敏感的任务；远端大模型处理复杂推理和罕见场景。

这个模式的关键路径是 memory。本地模型要真正谈得上”个性化”，需要持久化的跨会话上下文。远端模型要跨设备一致，也需要共享的记忆层。

但 memory 层目前没有明显领先的玩家。向量数据库、KV cache 持久化、结构化记忆文件、Hermes Agent 的自改进循环……各种方案并存。这块还没人跑出来——但谁先解决 memory，谁就拿到了代理持续性的钥匙。

验证器的空白

写代码有编译器和测试。做客服有满意度打分。但文档分析、策略制定、创意写作、数据分析报告——这些场景里，几乎没有靠谱的 verification 工具。

“你怎么知道 AI 给的结果是对的？”——这个问题在很多场景里至今没有好答案。没有验证工具，你就只能 blind trust。关键决策里，这不可接受。

验证器的机会很大。方向也很清楚：

  • 针对特定输出格式的结构化验证器
  • 基于规则的约束检查器
  • 模型间的交叉验证（用一个模型验证另一个）
  • 半自动的 human-in-the-loop 验证框架

谁能把验证器做出来，谁就打开了非编程场景的大门。

我设想的 agent 架构

Agent自主工作流里面的三层分工。

骨架是 workflow 模板、工具注册、路由惯例、安全基线。人画骨架不是画死框——模型当然可以偶尔越界，很多好结果就是这么来的。骨架给它方向，但不卡死它。

中间层是模型的工作区：代码、文案、分析、推理都在这一层。方向有了，怎么发挥是模型的事。

验证器才是真正兜底的。它的范围比骨架宽——不光检查输出是否符合骨架定义，还要做测试、交叉验证、tree shaking，确保任何越界行为不会变成问题。

在此之上的一层：meta工具层，用来系统性生成骨架和验证器，纯人类工作或者AI部分辅助。

市场推演：权力在转移

把上面几条线串在一起看：

模型供应商的定价权在收窄。 推理毛利率比什么都说明问题。Anthropic 每 1 美元收入要花 71 美分在推理算力上（Q1），Q2 降到了 56 美分。OpenAI 没有单独披露过推理毛利，但整体毛利率已经被推理成本压到 33%——2025 年推理支出翻倍到 $8.7B。Anthropic 刚递交 S-1，估值 $9650 亿。OpenAI 的 IPO 也在路上，2026 年预计亏损 $140 亿。

价值在往”用模型做什么”这一层转移。 当模型日益商品化，工具定义、workflow 编排、memory 管理、验证器——这些模块的供应商拿到的议价权会越来越大。Claude Code 的 $25 亿年化收入已经是一个信号。

B 端先跑通，C 端在后面。 企业愿意为确定性和可控性付费，所以 B 端率先落地。但推理成本还在降——DeepSeek 已经把 V4 Pro 的价永久砍了 75%——到时候面向消费者的 AI 服务会重新成为热门赛道。成本降下来，体验才能做上去。

最后

Scaling law 仍然成立。但更强的模型不等于更赚钱的生意。

如果你在找下一波投入的方向，这几个坐标比下一个 SOTA 模型更重要：harness、memory、verification、agent 软件。
